# Gestion des applications {#manage-applications}

Dans les déploiements d’expérience **une** application représente le service ou le produit que vous souhaitez contrôler à l’aide d’indicateurs de fonctionnalité. Avant que votre équipe puisse créer des indicateurs de fonctionnalité, vous devez intégrer au moins une application à la console.

## Qui peut gérer les applications ? {#permissions}

Seuls les membres dotés du rôle **Admin** peuvent ajouter ou modifier des applications. Si vous ne disposez pas de ce rôle, contactez l’administrateur ou l’administratrice de votre équipe.

## Ajouter une application {#add-application}

Pour ajouter une application à votre équipe, consultez [Intégration de votre application](onboard-your-application.md) pour obtenir des instructions détaillées.

## Propriété de l’équipe {#team-ownership}

Chaque application appartient à une seule équipe. Seuls les membres de cette équipe peuvent gérer les indicateurs de fonctionnalité de l&#39;application.

Ce modèle de propriété affecte également la façon dont vous structurez les groupes de fonctions :

* **Au sein de votre équipe** — Un propriétaire de version de produit peut regrouper les indicateurs de fonctionnalité dans les applications appartenant à la même équipe. Voir [Groupes de fonctionnalités pour contrôler plusieurs fonctionnalités](../../concepts/feature-groups-to-control-multiple-features.md).
* **Entre les équipes** — Si vous devez regrouper des indicateurs de fonctionnalités provenant d&#39;applications appartenant à différentes équipes, utilisez un groupe de fonctionnalités interéquipes, qui nécessite le rôle **Administrateur de fonctionnalités**.
