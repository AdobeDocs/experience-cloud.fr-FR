# Équipes et leurs administrateurs {#teams-and-admins}

Dans les déploiements d’expérience, une **équipe** est l’entité organisationnelle qui possède les applications et gère les indicateurs de fonctionnalité. Toutes les configurations d&#39;accès, d&#39;autorisations et d&#39;indicateurs de fonctionnalité sont limitées à une équipe.

## Structure de l&#39;équipe {#team-structure}

Les équipes dans les déploiements d’expérience ont une structure plate : tous les membres ayant le même rôle partagent les mêmes autorisations au sein de l’équipe. Il n’existe aucune hiérarchie au sein d’un rôle. Les différents rôles comportent différents niveaux d’accès. Voir [Rôles utilisateur](../teams/user-roles.md) pour une répartition complète.

Une équipe peut posséder plusieurs applications. Tous les indicateurs de fonctionnalité, groupes de fonctionnalités et versions de ces applications sont gérés dans le contexte de l&#39;équipe.

## Administrateurs d’équipe {#team-admins}

Chaque équipe compte un ou plusieurs **administrateurs**. Les administrateurs sont chargés de :

* Ajout et suppression de membres d’équipe
* Affecter et modifier des rôles utilisateur
* Valider ou rejeter les demandes d&#39;accès des nouveaux membres
* Gestion des applications détenues par l&#39;équipe

>[!TIP]
>
>Si vous ne savez pas qui est l’administrateur de votre équipe, consultez votre chef de produit ou d’ingénierie, ou contactez le responsable des déploiements d’expérience de votre entreprise.

## Trouver votre équipe {#finding-your-team}

Avant de demander l’accès, déterminez si une équipe existe déjà pour votre projet :

1. Vérifiez avec votre chef de produit ou d’ingénierie.
2. Vérifiez les équipes visibles dans la console Déploiements d’expérience si vous disposez déjà d’un accès limité.
3. Demandez à l’administrateur des déploiements d’expérience de votre organisation de vous fournir une liste des équipes actives.

S’il n’existe aucune équipe pour votre projet, vous devez en créer une. Voir [Créer une nouvelle équipe](create-a-new-team.md).
