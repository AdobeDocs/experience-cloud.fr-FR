# Intégration de déploiements d’expérience dans votre application {#integrate}

Cette section fournit des instructions d’intégration pour tous les clients Déploiements d’expérience, organisés par type d’application.

## Choisir le chemin d’intégration {#choose}

Sélectionnez le guide correspondant à votre type d’application :

1. [Guide de démarrage](startup-guide.md) — Commencez ici pour un aperçu général de toutes les étapes d’intégration
2. [Applications de bureau](desktop-applications.md) — Intégration directe de l’API REST pour les applications de bureau
3. [Applications mobiles](mobile-applications.md) — Intégration de l’API REST pour les applications mobiles
4. [Applications web](web-applications.md) — Intégration de l’API REST pour les propriétés web
5. [Services web](web-services.md) — Intégration SDK côté serveur pour les services principaux
6. [SDK](sdks.md) — Architecture de SDK, conditions préalables et SDK disponibles
7. [Étapes d’intégration](integration-steps.md) — Instructions d’intégration détaillées

## Types d’applications en un coup d’œil {#overview}

| Type d’application | Intégration recommandée |
|---|---|
| **Service web/serveur principal** | SDK Java SDK ou Node.js |
| **Application web** | API REST — API de fonctionnalités V3 |
| **Application mobile** | API REST — API de fonctionnalités V3 |
| **Application de bureau** | API REST — API de fonctionnalités V2 |
