# Contacter l’assistance {#contact-support}

Si vous avez besoin d’aide de l’équipe d’assistance des déploiements d’expérience, utilisez le canal d’assistance standard d’Adobe pour envoyer une demande. L’équipe d’assistance surveille les requêtes entrantes et répond le plus rapidement possible.

## Soumission d’une demande d’assistance {#submit-request}

Pour soumettre une demande d’assistance, procédez comme suit :

1. Connectez-vous à [](https://adminconsole.adobe.com).
2. Accédez à **Assistance** dans le volet de navigation supérieur.
3. Sélectionnez **Ouvrir un ticket d’assistance**.
4. Remplissez le formulaire de demande avec les détails de votre problème.
5. Attribuez le ticket au produit **Déploiements d’expérience** et envoyez-le.

Vous pouvez également utiliser l’option **Contacter l’assistance** disponible dans le panneau de droite de la console Déploiements d’expérience pour ouvrir une demande d’assistance directement à partir du produit.

## Éléments à inclure dans votre demande {#what-to-include}

Pour aider l’équipe d’assistance à enquêter sur votre problème aussi efficacement que possible, incluez les informations suivantes dans votre demande d’assistance :

* **Environnement** — Spécifiez si le problème se produit dans l&#39;environnement intermédiaire, de production ou les deux.
* **Indicateur de fonctionnalité ou version** — Indiquez le nom, la clé ou le lien direct vers l&#39;indicateur de fonctionnalité, le groupe de fonctionnalités ou la version concerné.
* **Tester les données utilisateur** — Incluez l&#39;identifiant utilisateur (e-mail, GUID ou identifiant visiteur) d&#39;un utilisateur qui rencontre le problème.
* **Procédure à suivre** — Décrivez les mesures exactes que vous avez prises pour résoudre le problème.
* **Comportement attendu ou réel** — Décrivez ce que vous vous attendiez à voir se produire et ce qui s&#39;est produit à la place.
* **Captures d’écran ou messages d’erreur** — Joignez les captures d’écran, les messages d’erreur ou les réponses de l’API qui illustrent le problème.
* **Tout autre contexte pertinent** — Incluez la version de SDK, l’ID du client de l’application ou les modifications récentes de la configuration qui peuvent être liées.

Plus vous fournissez de détails, plus l’équipe d’assistance peut diagnostiquer et résoudre le problème rapidement.
