# Gestion des équipes {#manage-teams}

Les équipes sont l’entité organisationnelle principale dans les déploiements d’expériences. Chaque équipe possède une ou plusieurs applications et gère tous les indicateurs de fonctionnalités, les groupes de fonctionnalités et les versions de ces applications.

Pour gérer une équipe, vous devez disposer du rôle **Admin**. Voir [Rôles utilisateur](user-roles.md) pour une description complète des rôles et autorisations disponibles.

## Tâches de gestion d&#39;équipe {#tasks}

En tant qu’administrateur d’équipe, vous pouvez effectuer les tâches suivantes :

* [Ajouter des membres à votre équipe](add-team-members.md) et leur attribuer le rôle approprié
* Supprimer les membres qui n&#39;ont plus besoin d&#39;un accès
* Approuver ou rejeter les demandes d’accès entrantes
* Intégrez de nouvelles applications à l’équipe. Voir [Gestion des applications](../applications/manage-applications.md)

## Structure de l&#39;équipe {#structure}

Les équipes dans les déploiements d’expérience ont une structure plate. Tous les membres ayant le même rôle partagent les mêmes autorisations : il n’existe aucune sous-hiérarchie au sein d’un rôle. Plusieurs rôles peuvent être affectés au même membre lorsqu’un accès plus large est nécessaire.

## Créer une nouvelle équipe {#create-team}

De nouvelles équipes sont créées sur demande. Voir [Création d’une équipe](../console/create-a-new-team.md) pour obtenir les informations requises et la procédure à suivre pour soumettre une demande.
